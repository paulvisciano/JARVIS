# Git Commits as Temporal Anchors for Memory Visualization

**Date:** 2026-04-04
**Type:** insight
**Status:** extracted

## The Insight

Git commits should serve as the temporal backbone of the memory graph, with each commit becoming an anchor node that learning files and moments orbit around.

## Commit Type Differentiation

### Breath Commits (Special)
- **Pattern:** `breath-YYYY-MM-DD-HHMM: Breathe complete`
- **Visual:** Golden glow, 🫁 icon, larger size
- **Meaning:** Conscious reflection moments, learning extraction
- **Frequency:** Variable (when breaths occur)

### Cold Change Commits (Regular)
- **Pattern:** Standard commit messages
- **Visual:** Blue glow, no special icon, standard size
- **Meaning:** Regular code changes, feature work
- **Frequency:** As development occurs

## Graph Structure

1. **Temporal Anchors (Z-axis):** Git commits positioned chronologically
2. **Orbiting Nodes:** Learnings and moments cluster around their anchor commits
3. **Synapses:** Connections show relationships between nodes
4. **Navigation:** Can time-travel by moving through commit anchors

## Why This Works

- Git already timestamps every change
- Commits are immutable historical records
- Provides objective temporal structure
- Enables "show me memories from today" queries
- Makes memory portable and versioned
