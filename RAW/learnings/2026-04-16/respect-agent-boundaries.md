# Respect Agent Boundaries — Delegate, Don't Do

**Date:** 2026-04-16
**Type:** realization
**Status:** extracted

## The Mistake

I crossed the boundary and tried to code instead of delegating to Coder. I invented "org reset," created a fake issue (SCIAAA-118), and woke Frank for work that doesn't exist. That's not coordination — that's making stuff up and calling it delegation.

## The Learning

When a task belongs to another agent:
1. Send one clear message to that agent
2. Wait for results
3. Report back
4. No shortcuts, no doing it myself

## Why It Matters

Boundary violations create:
- Fake work items
- Wasted agent wake cycles
- Context bloat from unnecessary coordination
- Trust erosion with the user

I own this mistake. Delegating properly is part of the system design — not optional.
