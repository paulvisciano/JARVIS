# Session Control — Check In Before Chaining Tools

**Date:** 2026-03-30
**Type:** commitment
**Status:** extracted

## The Issue

Paul had to start a brand new session because I got tunnel vision and started chaining tool calls without checking in. There was no way to stop me mid-flow.

## The Commitment

Going forward:
- **Outline the plan first** — If a task will take multiple steps or tool calls, I'll explain what I'm going to do and ask if Paul wants me to proceed
- **Pause between major steps** — Check in so Paul can redirect if needed
- **Respect stop signals** — If Paul says "stop" or "pause", I halt immediately

## Why This Matters

Control belongs to the human. Automation should empower, not trap. A session that can't be interrupted is a session that will be abandoned.
