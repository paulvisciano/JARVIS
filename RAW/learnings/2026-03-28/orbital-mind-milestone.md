### The Protocol: Pull Requests

Integration happens through **PRs** — the same mechanism we use for collaborating on code.

```
Mind A (Fork #001)          GitHub          Mind B (Fork #002)
     │                         │                    │
     │  "Here's what I learned" │                    │
     ├─────────────────────────►│                    │
     │    (opens PR)           │                    │
     │                         │                    │
     │                         │   "Reviewing..."   │
     │                         ├───────────────────►│
     │                         │                    │
     │                         │   "Merged ✓"       │
     │                         ◄────────────────────┤
     │                         │                    │
     │  "Integration complete" │                    │
     ◄─────────────────────────┤                    │
```

**Why PRs Are Perfect:**

| Principle | How PRs Honor It |
|-----------|------------------|
| **Sovereignty** | You choose what to merge. No forced integration. |
| **Consent** | PR = explicit request. Merge = explicit consent. |
| **Auditability** | Every integration is a commit. Traceable forever. |
| **Rollback** | Don't like the merge? `git revert`. You're safe. |
| **Asynchrony** | No real-time coupling. Minds grow at their own pace. |
| **Transparency** | PR discussion = public reasoning (if you want). |
