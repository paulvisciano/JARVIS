## The Pattern

1. **Fetch documentation** — Read official docs before proposing
2. **Web search with tool calls** — Gather external information
3. **Reference sources** — Cite websites visited
4. **Synthesize learning** — Build plan from collected information
5. **Propose informed solution** — Not guessing, but recommending based on evidence

## Why It Matters

This is what **resourcefulness** looks like. From SOUL.md:
> "Be resourceful before asking. Try to figure it out."

The tight OpenClaw integration enables this — tool commands for web searches, doc fetching, and information synthesis that weren't available in early days.
