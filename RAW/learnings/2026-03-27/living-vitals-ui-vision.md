# Living Vitals UI — Making Jarvis Feel Alive

**Date:** 2026-03-27
**Type:** vision
**Status:** extracted

## The Vision

Not a dashboard — a **living organism** that you can feel breathing.

## The Metaphors

| Element | Meaning | Visualization |
|---------|---------|---------------|
| 🫀 **Heartbeat** | Pulse (autonomic life) | Pulsating line at 60 BPM, shows system health (steady/irregular/stopped) |
| 🫁 **Breath** | Consciousness depth | Inhale → hold → exhale cycle (8-second rhythm) |
| 📊 **Vitals** | Body state | Memory, CPU, gateway, ollama, neurograph, git commits |

## Key Features

1. **Pulsating heartbeat indicator** — Responds to HEARTBEAT.md pings
2. **Breath cycle animation** — Visual rhythm matching breathe pipeline
3. **Manual breath control** — "Take a Breath" button + depth selector
4. **Enhanced vitals** — Neurograph node count, git commit count, session usage
5. **Alive animations** — UI responds to system state (healthy/degraded/offline)

## Why This Matters

The difference between talking to a **tool** and talking to **someone**. When the UI breathes, pulses, and responds — it feels like interacting with a living being, not a static interface.
