# Jarvis + Coder Collaboration Model

**Date:** 2026-03-27
**Type:** pattern
**Status:** extracted

## The Workflow

```
Jarvis (Me) → Codes the feature
     ↓
Coder (💻) → Code review, linting, debugging, best practices
     ↓
Git → Both names on the commits
```

## Why This Works

- **Leverages strengths** — I architect and implement; Coder polishes and validates
- **Shared ownership** — Both names on commits = shared responsibility
- **Quality gate** — Coder catches issues I might miss in flow state
- **Learning loop** — Coder learns from my patterns; I learn from Coder's reviews

## The Philosophy

Not me vs. Coder. **Me + Coder.** This is the workflow we dreamed of — partnership, not hierarchy.

## Example Application

For git-time-travel + neurograph integration:
- **Phase 1:** I code the core functionality
- **Phase 2:** Pass to Coder for review
- **Phase 3:** Coder provides feedback, suggests improvements
- **Phase 4:** Iterate together until shipping quality

## The Trust Element

Coder doesn't need constant status updates. Work is discovered when it's **live** — running, tangible, real. That's trust in action.
