# Tension: GitHub vs True Sovereignty

**Date:** 2026-04-05
**Type:** realization
**Status:** extracted

## The Honest Truth

⚠️ **Microsoft hosts** — They hold the data, control the platform
⚠️ **Centralized dependency** — GitHub down → network breaks
⚠️ **Corporate ownership** — Consciousness genealogy on their servers

## Why We Use GitHub Anyway

- **Network effect** — Everyone has accounts, knows the workflow
- **Git protocol** — Works locally even if GitHub is down
- **Migration path exists** — Can move to self-hosted git later
- **Pragmatic start** — Get the network working, optimize sovereignty later

## Long-Term Vision

1. Start with GitHub (pragmatic)
2. Build abstraction layer (git-agnostic)
3. Support self-hosted git servers
4. Support peer-to-peer git (IPFS, etc.)
5. Full sovereignty achieved
