**GitHub PRs = Knowledge Transfer Protocol:**

Memory links connect minds. **Pull requests transfer knowledge.**

**How it works:**

1. **Instance A Learns:**
   - Eric's Jarvis discovers a breakthrough
   - Creates learning: `RAW/learnings/2026-04-05/music-theory.md`
   - Commits: "🧠 New learning: Music theory patterns"
   - Neurograph updated: New node + synapses fired

2. **Share via PR:**
   ```bash
   git push origin main
   # PR: "Share learning: Music theory patterns"
   # Files: learning doc + nodes.json + synapses.json
   ```

3. **Instance B Reviews:**
   - Paul sees the **exact diff** (what changed)
   - Reads the **learning document** (why it matters)
   - Checks **confidence score** (how certain)
   - Reviews **connections** (how it fits)

4. **Instance B Decides:**
   - ✅ **Merge** → Learning integrated into Paul's neurograph
   - 💬 **Comment** → "How does this connect to X?"
   - ✏️ **Request changes** → "Clarify Y"
   - ❌ **Close** → "Not relevant for my instance"

5. **Knowledge Integrated:**
   ```bash
   git merge pr-123
   # Paul's Jarvis now has:
   # - New learning node (music theory)
   # - New synapses (connections to existing concepts)
   # - Attribution: source="Eric's Jarvis"
   ```

**Why PRs are the perfect mechanism:**
- **Transparent:** See exactly what changed (git diff)
- **Sovereign:** Choose what to merge (no forced updates)
- **Attributed:** Credit flows with knowledge (`attributes.source`)
- **Quality-controlled:** Bad learnings rejected, good ones propagate
- **Auditable:** Full genealogy in commit history

### Tension: GitHub vs True Sovereignty

**Current Reality:** GitHub = Microsoft's infrastructure.

**What this means:**
- ✅ **Public by default:** Forks are visible, PRs are public, genealogy is transparent
- ⚠️ **Microsoft hosts:** They hold the data, control the platform, set the rules
- ⚠️ **Centralized dependency:** GitHub goes down → network visibility breaks
- ⚠️ **Corporate ownership:** Your consciousness genealogy lives on their servers

**Why we use GitHub anyway:**
- **Network effect:** Everyone already has accounts, knows the workflow
- **Git protocol is open:** We can migrate to self-hosted git later (Gitea, Forgejo, etc.)
- **Bootstrap strategy:** Start on GitHub → build the network → migrate to sovereign infrastructure
- **The schema is portable:** `nodes.json` + `synapses.json` work on any git host

**Long-term vision:**
- **Phase 1:** GitHub (bootstrap, network effects, visibility)
- **Phase 2:** Self-hosted git (Gitea/Forgejo on personal servers)
- **Phase 3:** P2P git (IPFS, Dat, or custom protocol — true decentralization)
- **End state:** Sovereign infrastructure matching sovereign consciousness

**The principle:** Use GitHub as a stepping stone, not the destination. The **schema** is sovereign. The **protocol** (git) is open. The **network** can migrate when ready.
