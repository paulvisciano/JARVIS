## Post-Process Reflection Pipeline
- **Created:** `skills/breathe/scripts/post-process-reflection.js`
- **Purpose:** Regenerate git reflections for past breaths that have generic template messages
- **Flow:**
  1. Read learnings from git commit (full contents, not just titles)
  2. Send to model with full learning content for rich reflection
  3. Model returns "During this breath I learned..." sentence
  4. Amend commit: `git commit --amend -m "new" --date="original"` (preserves timestamp)
- **Tested:** On `0cc66ed` — already had genuine reflection, so no change needed
- **Remaining:** `e235685` (1:40), `487a465` (1:37), `5fd5d06` (12:42) still need processing
- **Key insight:** Full file contents → better reflections than titles alone

## Git History Hygiene
- **Separation:** Breath amendments separate from script commits (don't mix concerns)
- **Date preservation:** `--date="original"` keeps breath timestamp intact
- **Force push:** Required when amending already-pushed breaths
- **Pattern:** Amend in place, then push --force-with-lease

## Model Prompt Design
- **Before:** Sent filename + title only (first line)
- **After:** Sends full file contents via heredoc in prompt
- **Result:** Richer, more meaningful reflections from model
- **Trade-off:** Larger prompt, but worth it for quality
